Linux文件与目录管理
发表于 2019-11-06| 更新于 2019-11-26
字数总计: 2.8k|阅读时长: 8 分钟|阅读量:

目录相关的操作

一些常用的操作

cd：Change Directory

pwd:Print Working Directory

mkdir:make directory

rmdir:remove directory

关于执行文件路径的变量:$PATH

​ 可以通过 echo $PATH命令来看到底有哪些命令被定义出来了

​ 可以通过如下命令将目录/root加入PATH中： PATH="${PATH}:/root"

文件与目录的管理

​ 在文件与目录的管理上，不外乎显示属性，拷贝，删除文件和移动文件或目录这几种操作

  • 文件与目录的检视：ls

  • 复制，删除与移动：cp,rm,mv

    要复制文件，使用cp即可，不多cp这个命令还可以建立连结档(就是快捷方式)，对比来那个文件的新旧而给予更新，以及复制整个目录等功能，而mv命令也可以直接拿来做更名动作

​ 这个cp 的功能有很多，由于我们经常会进行复制操作，我们如果去复制别人的数据(当然，该文件必须有read权限才可以)，总是希望复制到的数据最后是我们自己的，所以在预设的条件中，cp的来源档与目的档的权限是不同的，目的档的拥有者通常是指令操作者本身，由于具有这个特性，因此当我们在进行备份的时候，某些需要特别注意的特殊权限文件 ，例如密码文件(/etc/shadow)以及一些配置文件，就不能直接以cp进行复制，必选加上-a或-p等可以完整复制的选项才行

​ 实体链接和符号链接：

​ 这个比较有趣，使用-l与-s都会建立所谓的连结档(link file)，但是这两种不一样，-l是实体链接(hard link),至于-s是符号链接(symbolic link),简单说。bashrc_slink是一个快捷方式，这个快捷方式会连结到bashrc去，所以档名右侧会有指向(->)的符号，至于bashrc_hlink文件与bashrc的属性与权限完全一模一样，与尚未连结钱的差异则是第二栏的link数由1变成2了

需要注意的是使用mv命令是如果有多个来源文件或目录，则最后一个目标文件一定是目录，意思是说将所有的数据移动到该目录的意思例如：mv test1 test2 test3

文件内容查阅

  • cat 由第一行开始显示文件内容
  • tac 从最后一行开始显示
  • nl 显示的时候，顺道输入行号(cat加-n也可以显示)
  • more 一页一页的显示文件内容，可以用/+字符串搜索内容，q退出more
  • less和more相似，不过他可以向前翻页
  • head 只看头几行
  • tail 只看尾巴几行
  • od 以二进制的方式读取文件内容

修改文件时间或建置新档：touch

  • 文件时间主要有三个：

    • modification time(mtime)

      当该文件的[内容数据]变更时，就会更新这个时间，内容数据指的是文件的内容，而不是文件的属性或权限

    • status time(ctime)

      当该文件的[状态]改变时，就会更新这个时间，举例来说，像是权限与属性被更改了，都会更新这个时间

    • access time(atime)

      当[该文件的内容被取用]时，就会更新这个读取时间(access),举例来说，我们使用cat去读取/etc/man_db.conf,就会更新该文件的atime了

文件预设权限

​ 当你建立一个新的文件或者目录时，它的默认权限是什么呢？他和umask有关，umask就是指定[目前用户在建立文件或目录时候的权限默认值]

需要注意的是，在默认权限的属性上，目录与文件是不一样的，一般文件建立不应该有执行的权限，因为一般文件通常就是用在数据的记录，当然不需要有执行的权限：

  • 若使用者建立为[文件]则预设[没有可执行(x)权限]，亦即只有rw这两个项目，预设权限-rw-rw-rw-
  • 若用户建立为[目录]，则由于x与是否可以进入此目录有关，因此默认为所有权限均开放预设权限为drwxrwxrwx

但是，umask的分数指的是[该默认值需要减掉的权限]例如拿掉读(r)的权限就是4分。

如果以上面的例子来说，因为umask为022，所有user并没有被拿掉任何权限，不过group与others的权限被拿掉2,也就是读(w)的权限，那么多当使用者：

  • 建立文件时:(-rw-rw-rw-) - (———w—w—) =>-rw-r—r—
  • 建立目录时:(drwxrwxrwx) - (d——w—w—) =>drwxr-xr-x

文件的特殊权限：SUID,SGID,SBIT

文件处理rwx之外，还有其他特殊的权限

  • Set UID

    当 s 这个标志出现在文件拥有者的 x 权限上时，此时就被称为Set UID,那么SUID 的权限对于一个文件的特殊功能是什么呢？基本上SUID有这样的限制于功能：

    • SUID权限仅对二进制程序有(binary program)效

    • 执行者对该程序需要具有 x 的可执行权限

    • 本权限仅在执行该程序的过程中有效(run-time)

    • 执行者将具有该程序拥有者(owner)的权限

      例如在Linux系统中，所有的账号都是记录在/etc/shadow这个文件内，这个文件的权限为：[—————1 root root],意思是这个文件仅有root可读且仅有root可以强制写入而已，然而我们使用自己的账号(LemonLIU)是可以修改自己密码的，这就是SUID的功能了：

      1. LemonLIU对于/usr/bin/passwd 这个程序具有x 的权限，表示LemonLIU能执行passwd
      2. passwd的拥有者是root
      3. LemonLIU执行passwd的过程中，会暂时获得root权限

      4. /etc/shadow 就可以被LemonLIU所执行的passwd所修改

        而使用cat就不能读取/etc/shadow，因为cat不具有SUID权限

  • Set GID

    当 s 标志出现在文件拥有者的 x 项目为SUID,那么 s 在群组的 x 时称为Set GID,与SUID 不同的是，SGID可以针对文件或目录来设定，如果对于文件来说，SGID有如下的功能：

    • SGID对二进制程序有用
    • 程序执行者对该程序来说，需具备 x 权限

    • 执行者在执行的过程中将会获得该程序群组的支持

      对于目录来说有如下功能：

    • 用户对于此目录具有r与x 的权限时，该用户能进入此目录

    • 用户在此目录下的有效群组(effective group)将会变成该目录的群组
    • 用途：如用户在此目录下具有w的权限(可以新建文件)，则使用者所新建的文件，该文件的群组与此目录的群组相同

  • Sticky Bit

    这个Sticky Bit,SBIT目前只针对目录有效，对于档案已经没有效果了，SBIT对目录的作用是：

    • 当用户对此目录具有w，x权限，亦即具有写入的权限时

    • 当用户在该目录下建立档案或目录时，仅自己与root才有权利删除该档案

      换句话说：当甲这个用户对于A目录是具有群组或其他人身份，并且拥有该目录 w 的权限，这表示[甲用户对该目录内任何人建立的目录或档案均可进行”删除/更名/搬移”等动作]，不过，如果A目录加上看SBIT的权限项目时，则甲只能够针对自己建立的档案或目录进行删除/更名/移动等动作，而无法删除他人的档案

      ​

SUID/SGID/SBIT权限设定

那么如何配置文件是成为具有SUID与SGID的权限呢？这就需要数字更改权限的方法了，在原有的三个数字的权限组合前再加一个数字，最前面的这个数字就代表这几个权限了

  • 4为SUID
  • 2为SGID
  • 1为SBID

最后一个主意，因为666没有 x 的权限，所以S与T代表空的

观察文件类型：file

如果你想知道某个档案的基本数据，例如是ASCII或者data档案，就可使用file来检阅

指令与档案的搜寻

  • 脚本文件名的搜寻

    which(寻找执行档)

    我们知道在终端机模式当中，连续输入两次[tab]键就能知道用户有多少指令可以下达，那么如歌查看这些指令的完整文件名放在那？可以使用which或type来查找

​ 这个指令是根据[PATH]这个环境变量规范的路径去搜寻[执行档]的档名，cd找不到的原因是cd是[bash内建的指令]

  • 档案档名的搜寻

    搜索档名通常不会使用find，因为速度慢之外，很使用硬盘，通常我们都是先使用whereis或者是locate来检查，因为whereis和locate都是利用数据库搜寻数据没有直接搜寻硬盘，省时间

    • whereis(寻找特定档案)

    • locate

      这个locate更简单直接在后面输入[档案的部分名称]后，就能得到结果，他的限制是因为locate寻找的数据是由[已建立的数据库/var/lib/mlocate/]里面的数据搜寻的，而数据库的建立没默认是每天执行一次，所有当你新建立起来的档案，却还在更新之前搜寻该档案就找不到，手动更新数据库命令[updatedb]

  • find

需要注意的是上面的find / -prem +7000在 findutils-4.5.12中已经改为了find / -prem /7000

权限与指令间的关系

  1. 让用户能进入某目录称为[ 可工作目录 ]的基本权限：
    • 可使用的指令：例如cd等变换工作目录的指令
    • 目录所需权限：用户对这个目录至少需要具有x权限
    • 额外需求：如果用户想要对这个目录进行ls查询，至少具有r权限
  2. 用户在某个目录内读取一个档案的基本权限：
    • 可使用指令：例如本章谈到的 cat, more, less等等
    • 目录所需要的权限：用户对这个目录至少需要 x 权限
    • 档案所需权限：使用者对档案至少具有r权限
  3. 让使用者可以修改一个档案的基本权限
    • 可使用的指令：vi编辑器等
    • 目录所需权限：用户对这个档案所在的目录至少具有x权限
    • 档案所需权限：使用者对该档案至少有r，w权限
  4. 让一个使用者可以建立一个档案的基本权限
    • 目录所需权限：用户对该目录具有w，x权限
  5. 让用户进入某目录并执行该目录下的某个指令的基本区权限
    • 目录所需权限：用户在该目录至少具有x的权限
    • 档案所需权限：使用者在该档案至少需要有x的权限
