Django处理请求过程及其MTV开发模式
发表于 2019-11-01| 更新于 2019-11-26
字数总计: 319|阅读时长: 1 分钟|阅读量:

Django处理一个请求的过程：

  1. Django使用的是根 URLconf 模块，这个值是通过 ROOT_URLCONF设置(在…/settings.py中)
  2. Django加载 URLconf 模块(urls.py文件)，并寻找可用的urlpatterns
  3. Django依次匹配每个URL模式，在于请求的URL匹配的第一个模式处停下来
  4. 一旦其中的一个正则表达式匹配上，则Django将请求指向对应的视图函数处理
  5. 如果没有匹配到正则表达式，或者过程中抛出一个异常，则Django将调用一个适当的错误处理视图

MTV开发模式

​ 由于C由框架自行处理，而Ddango里更关注的是模型(Model),模板(Template)和视图(Views),因此Django也被称为MTV框架，在MTV开发模式中：

  • M代表模型(Model),即数据存储，该层处理与数据相关的所有业务，即如何存取，如何验证有效
  • T代表模板(Template),即表现层，该层处理与表现相关的决定，即如何在页面或其他类型文档中进行显示
  • V代表视图(View),即业务逻辑层，该层包含存取模型及调取恰当模板的相关逻辑，可以将他看做模板与模型之间的桥梁
