BASH的总结
发表于 2019-11-20| 更新于 2019-11-26
字数总计: 712|阅读时长: 2 分钟|阅读量:

Bash shell的功能

  • 命令编修能力(history)

    他能记录使用过的命令，记录在家目录的.bash_history,需要注意的是记录的是上一次登入以前执行的命令，至于本次登录的命令都记录在内存，注销系统才会被记录到文件中

  • 命令与文件的补全功能

    • [ Tab ]接在一串指令的第一个字的后面，则为命令补齐
    • [ Tab ]接在一串指令的第二个字后面是，则为文件补齐

  • 命令别名设定的功能(alias)

    alias lm ='ls -al'

  • 工作控制，前景背景控制

  • 程序化脚本(shell scripts)

  • 通配符(Wildcard)

删除已输入的命令或移动光标的快捷键

变量

  • 变量的设定，使用[=]连接变量和他的内容即可
  • 变量的显示，使用echo $ 变量

变量设定规则

  • 变量与变量内容以[=]连接，等号两边不能直接接空格
  • 变量名称只能是英文字母与数字，但是开头字符不能是数字
  • 变量内容若有空格可用[“”]或[‘’]将变量内容结合起来，但需要注意的是：
    • 双引号内的特殊字符如$等可以保持原有的特性
    • 单引号内的特殊字符则仅为一般字符(纯文本)
  • 可用跳脱字符[ \ ]将特殊字符(如Enter，$, 空格符等)变成一般字符
  • 若该变量需要在其他子程序执行，则需要以export来使变量变成环境变量：[export PATH]
  • 取消变量的方法为使用unset：[unset + 变量名称]

bash的进站与欢迎讯息

  • 进站画面显示/etc/issue

  • 欢迎文字/etc/motd里面修改即可

通配符与特殊符号

除此之外常用的bash环境的特殊符号还有：

  • #:批注符号，后面均不执行
  • |：分隔两个管线的界定
  • ;：连续指令下达分隔符
  • >,>>：数据流重导向，输出导向，分别为取代与累加
  • >,>>: 输入导向

grep

grep的作用是分析一行讯息，其中有我们需要的信息就取出来

排序命令

  • sort

  • uniq：如果排序完了想要将重复的资料仅显示一个

    • -i:忽略大小写字符的不同
    • -c:进行计数

  • wc：计算一个文件的行数，字数(英文单数)，字符数

    • -l：仅列出行
    • -w：仅列出多少字
    • -m：多少字符

字符转换命令

  • tr：可以用来删除一段讯息中的文字或者进行文字信息的替换

    • -d:删除讯息当中命令后面跟的字符串
    • -s：取代掉重复的字符

  • col

    • -x：将tab键转换成对等的空格键
    • -A：将tab键转为^I符号

  • paste：将两行贴在一起，且中间以tab键隔开
