MySQL基础与进阶

MySQL的基础与进阶

  1. SQL的概念

    Structured Query Language 结构化查询语言

  2. SQL语句分类

    1) Data Definition Language (DDL 数据定义语言) 如：建库，建表

    2) Data Manipulation Language(DML 数据操纵语言)，如：对表中的记录操作增删改

    3) Data Query Language(DQL 数据查询语言)，如：对表中的查询操作

    4) Data Control Language(DCL 数据控制语言)，如：对用户权限的设置

  3. MySQL的语法

    1. 每条语句以分号结尾
    2. SQL中不区分大小写，关键字中认为大写和小写是一样的
    3. 3种注释、
      • —空格 ：单行注释
      • / / : 多行注释
      • # ： MySQL特有的注释方式

  4. DDL操作数据库

    1. 创建数据库的几种方式

      • 创建数据库：CREATE DATABASE 数据库名；
      • 判断不存在则创建：CREATE DATABASE IF NOT EXISTS 数据库名；
      • 创建数据库并指定字符集：CREATE DATABASE 数据库名 CHARACTER SET 字符集

    2. 查看数据库

      show databases；

    3. 查看某个数据库的定义信息

      show create database test;

    4. 修改数据库默认的字符集

      ALTER DATABASE 数据库名 DEFAULT CHARACTER SET 字符集;

    5. 删除数据库

      DROP DATABASE 数据库名

    6. 查看正在使用的数据库

      SELECT DATABASE();使用的一个mysql中的全局函数

    7. 使用/切换数据库

      USE 数据库名；

    8. 创建表的格式

      CREATE TABEL 表名{

      ​ 字段名1 字段类型1，

      ​ 字段名2 字段类型2

      }；

    9. 数据类型

      • 整数 int(integer) 整型：整数类型(占 32 位二进制)
      • 小数
        • float 单精度浮点数，占 4 个字节
        • double 双精度浮点数，占 8 个字节
      • 日期
        • time 表示时间类型
        • date 表示日期类型
        • datetime 同时可以表示日期和时间类型
      • 字符串
        • char(m) 固定长度的字符串，无论使用几个字符都占满全部，M 为 0~255 之间的整数
        • varchar(m) 可变长度的字符串，使用几个字符就占用几个，M 为 0~65535 之间的整数
      • 文本
        • tinytext 允许长度 0~255 字节
        • text 允许长度 0~65535 字节
        • mediumtext 允许长度 0~167772150 字节
        • longtext 允许长度 0~4294967295 字节

    10. 查看某个数据库中所有的表

      show tables;

    11. 查看表结构

      DESC 表名

    12. 查看创建表的SQL语句

      SHOW CREATE TABLE 表名；

    13. 创建一个表结构相同的表

      create table 新表名 like 旧表名

    14. 直接删除表

      drop table 表名

    15. 判断存在则删除

      drop table if exists 表名

    16. 添加表列：ADD

      alter table 表名 ADD 列名 类型

    17. 修改表列类型：modify

      alter table 表名 modify 列名 新的类型；

    18. 修改列名：change

      alter table 表名 change 旧列名 新列明 类型；

    19. 删除列drop

      alter table 表名 drop列名；

    20. 修改表名

      rename table 表名 to 新表名

    21. 修改字符集

      alter table 表名 character set字符集；

  5. DML操作表中的数据

    1. 插入全部字段

      • 将所有的字段名都写出来

        insert to 表名(字段名1，字段名2，字段名3...)values(值1，值2，值3...)

      • 不写字段名

        insert into 表名 values(值1，值2，值3...)

    2. 插入部分数据

      insert into 表名(字段名1，字段名2)values(值1，值2)；

    3. 注意事项

      字符和日期型数据应包含在单引号中。MySQL 中也可以使用双引号做为分隔符

    4. DOC端乱码问题的解决

      • 查看包含character开头的全局变量

        show variables like 'character%'

      • 修改client，connection，results的编码为GBK，保证和DOS命令行编码一致

      • 单独设置

        set character_set_client=gbk; 修改客户端的字符集为 GBK

        set character_set_connection=gbk; 修改连接的字符集为 GBK

        set character_set_results=gbk; 修改查询的结果字符集为 GBK

      • 同时设置三项

        set names gbk；

    5. 将一张以及存在的表中的数据复制到另一张表中

      insert into 表名1 select * from 表名2

    6. 将一张以及存在的表中的部分数据复制到另一张表中

      insert into 表名1(列1，列2)select 列1，列2 from 表名2；

    7. 更新表记录

      • 不带条件修改数据

        update 表名 set 字段名=值；

      • 带条件的修改数据

        update 表名 set 字段名=值 where 字段名=值

      • 不带条件删除数据

        delete from 表名

      • 带条件删除数据

        delete from 表名 where 字段名=值

      • 使用truncate删除表中所有记录

        truncate table 表名

        truncate 相当于删除表的结构，再创建一张表

  6. DQL查询表中的数据

    1. 查询表所有行和列的数据

      select * from 表名

    2. 查询指定列的数据，多个列之间用逗号分隔

      select 字段名1，字段名2，字段名3，...from 表名

    3. 对列指定别名

      select 字段名1 AS 别名，字段名2 AS 别名...from 表名；

    4. 对列和表同时使用别名

      SELECT 字段名 1 AS 别名, 字段名 2 AS 别名... FROM 表名 AS 表别名;

    5. 查询指定列并且结果不出现重复数据

      select distinct 字段名 from 表名；

    6. 查询结果参与运算

      • 某列数据和固定值运算

        select 列名1+固定值 from 表名

      • 某列数据和其他列数据参与运算

        select 列名1 + 列名2 from 表名

      • 注意参与运算的必须是数值类型

    7. 条件查询

      • <、>、<=、>=、=、<> ，<>在 SQL 中表示不等于，在 mysql 中也可以使用!= 没有==
      • between…and:在一个范围内，包括首尾
      • in（集合）：集合表示多个值，使用逗号分隔
      • like’张%’：模糊查询
      • is null：查询某一列为null的值，注意不能写=null

    8. 逻辑运算符

      • and 或&&：与 ，建议使用前者，后者不通用
      • or 或 || : 或
      • not 或 ！ ： 非

    9. MySQL通配符

      • %：匹配任意多个字符串
      • _:匹配一个字符
