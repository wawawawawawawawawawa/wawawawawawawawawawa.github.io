Linux磁盘与文件系统
发表于 2019-11-08| 更新于 2019-11-26
字数总计: 2.2k|阅读时长: 6 分钟|阅读量:

  • ext2格式化后的启动扇区信息以及区块群组信息
  • 文件系统将block格式化为1k后最大容纳档案为16GB，与inode的限制一致

扇区，磁柱

  • 扇区(Sector)为最小的物理存储单位，每个扇区为512bytes
  • 将扇区组成一个圆，那就是磁柱(Cylinder)，磁柱是分隔槽(partition)最小的单位
  • 第一个扇区最重要，里面有主要开机区(Master boot record,MBR)以及分隔表(partition table)，其中MBR站446bytes，而partition table 则占有64bytes

文件系统特性

​ 我们都知道磁盘分区完毕后还需要进行格式化，之后操作系统才能后使用这个分隔槽，这是因为每种操作系统所设定的文件属性/权限并不相同，为了存放这些档案所需的数据，因此就需要将分隔槽进行格式化，以成为操作系统能够利用的文件系统格式(filesystem)

​ 传统的磁盘与文件系统的应用中，一个分隔槽就只能被格式化成为一个文件系统，所以我们可以说一个filesystem就是一个partition，由于新技术的利用，例如我们常听到的LVM与软件磁盘阵列(software raid),这些技术可以将一个分隔槽格式化为多个文件系统(例如LVM)，也能够将多个分隔槽合成一个文件系统(LVM,RAID),所以我们在格式化时已经不再说成针对partition来格式化了，通常我们可以称呼一个可被挂载的数据为一个文件系统而非分隔槽
文件系统的运作与操作系统的档案数据有关，较新的操作系统的档案数据除了档案袋实际内容外，通常还有非常多的属性，例如Linux操作系统的档案权限(rwx)与文件属性(拥有者，群组，时间参数等)，文件系统通常会将这两部分的数据分别存放在不同的区块，权限与属性放置到inode中，至于实际数据则放置到data block中，另外还有一个超级区块(superblock)会记录整个文件系统的整体信息，包括inode与block的总量，使用量等

ext2与ext3

  • ext2采用 索引是文件系统(indexed allocation)
  • ext3采用日志式文件系统(Journaling filesystem)

Linux文件系统的运作

因为数据要加载到CPU后才能处理，一个很大的档案在编辑过程中需要频繁的要系统来写入磁盘，由于磁盘写入的速度比内存慢很多，会耗在读写上造成低效率，Linux的解决方法为采用异步处理(asynchronously),所谓的异步处理是这样的：

​ 当系统加载一个档案到内存后，如果该档案没有被更动过，则在内存区段的档案数据会被设定为干净(clean)的，但如果内存的档案数据被更改过了，此时该内存的数据会被设定为脏的(Dirty),此时所有的动作在内存中，系统会不定时的将内存中设定为(Dirty)的数据写回到磁盘，以保持磁盘与内存中的数据一致，也可以使用sync指令手动强迫写入磁盘

其他Linux支持的文件系统与VFS

虽然Linux的标准文件系统是ext2，且还有增加了日志功能的ext3，事实上，Linux还有支持很多文件系统格式的，尤其是最近这几年退出了好几种速度很快的日志文件系统，包括SGI的XFS文件系统，以及Windows的FAT文件系统等

支持当前Linux支持的文件系统：ls -l /lib/modules/$(uname -r)/kernel/fs

系统目前已加载到内存中支持的文件系统：cat /proc/filesystems

文件系统的简单操作

  • 磁盘与目录的容量

    磁盘的整体数据是在superblock区块中，但是每个个别档案的容量则在inode当中记录，文字接口下查询一般用一下的两个命令：

    • df：列出文件系统的整体磁盘使用量
    • du：评估文件系统的磁盘使用量(常用在推估目录所占容量)

​ 因为df主要读取的数据几乎都是针对一整个文件系统，因此读取的范围主要是在Superblock内的信息，所以这个指令显示结果的速度非常的快速

​ 需要注意的是，使用-a 的参数时，系统会出现/proc这个挂载点，但是里面的数据都是0，因为/proc的东西都是Linux系统加载的系统数据，而且是挂载在内存的，所以不占硬盘空间

​ 至于/dev/shm/目录，其实是利用内存虚拟出来的磁盘空间，由于是透过内存仿真出来的磁盘，因此你在这个目录底下建立任何数据文件是，访问数据都是非常快速的，但是建立的东西再下次开机就消失了，因为在内存

与df不同的是，du这个指令其实会直接到文件系统内去搜寻所有的档案大数据，此外在默认的情况下，容量的输出是以KB来设计的，如果你想要知道目录占了多少MB，加入-m参数即可，想要知道目录占了多少容量，加入-s即可

实体链接与符号链接：ln

在Linux底下的连结档有两种，一种是类似Windows的快捷方式功能的档案，可以让你快速的连接到目标档案(或目录)；另一种则是透过文件系统的inode连结来产生新档名，而不是产生新档案，这种称为实体链接(hard link)

  • Hard Link(实体链接，硬式连结或实际连结）

    • 每个档案都会占用一个inode，档案内由inode的记录来指向

    • 想要读取该档案，必须要经过目录记录的文件名来指向正确的inode号码才能读取

      也就是说，其实文件名只与目录有关，但是档案内容则与inode有关，那么有没有可能多个档名对应的到同一个inode号码呢，有的，这就是hard link的由来，这样不论你使用哪个档名来编辑，最终结果都会写入到相同的inode与block中，因此均能进行数据的修改，一般来说，使用hard link设定链接文件时，磁盘空间与inode数目都不会改变，只是某个目录的block多写入一个关联数据而已

  • Symbolic Link(符号链接，亦是快捷方式)

    Symbolic link就是再建立一个独立的档案，而这个档案会让数据的读取指向他link的那个档案的档名，由于只是利用档案来作为指向的动作，所以，当来源档被删除之后，symbolic link的档案会打不开

建立连结档的ln指令

由于Hard Link的限制太多了，所以现实中一般会是Symbolic Link的使用较多

如果ln使用-s参数，就做成了Windows底下的快捷方式的意思，当修改Linux下的symbolic link档案时，其实改变的是原始档，此外如果做例如ln -s /bin/root/bin的连结时，将会连结bin目录，这是修改链接目录将会修改原始目录

关于目录的link数量

当我们建立目录时，因为一个空目录里至少会有三个东西，例如建立/tmp/testing,会有/tmp/testing,/tmp/testing/.与/tmp/testing/.. ，而前两个是一样都代表该目录，而最后一个则代表/tmp这个目录，所以说当我们建立一个新目录时，新的目录的link数为2，而上层目录的link数则会增加1
