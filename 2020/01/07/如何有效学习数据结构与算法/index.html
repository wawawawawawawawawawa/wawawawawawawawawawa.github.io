如何有效学习数据结构与算法
发表于 2020-01-07| 更新于 2020-01-08
字数总计: 570|阅读时长: 1 分钟|阅读量:

​ 很多时候会去想熟悉一个未知的领域需要付出什么代价，会犹豫自己找到的是不是最好的资料，会不会是在浪费时间，其实没有必然合适的，那为何不坚持尝试完手里自认为最优的呢，试着坚持完！

如何精通一个领域

•Chunk it up 切碎知识点
•Deliberate Practicing 刻意练习
•Feedback 反馈

​ 出自一本书，异类，著名的一万小时定律也是出自此处，要想学会数据结构与算法，应该将模块细分，然后刻意的强化练习，在练习中得到反馈以弥补自己的不足，反馈包括主动式的反馈和被动式的反馈，主动式反馈来源于主动去看高手的代码，学习强化自己的代码，被动式反馈来自去找高手看自己的代码，找到自己的不足，补齐他

数据结构

  • 一维

    • 基础：数组array(string),链表linked list
    • 高级：栈stack，队列queue，双端队列deque，集合set，映射map(hash or map)，etc

  • 二维

    • 基础：树tree，图graph
    • 高级：二叉搜索树binary search tree(red-black tree,AVL)，堆heap，并查集disjoint set，字典树Trie，etc

  • 特殊

    • 位运算Bitwise，布隆过滤器BloomFilter

    • LRU Cache

      需要注意的是最后需要总结为一张包含每个数据结构的原理和代码框架的脑图

算法

  • if-else,switch -> branch
  • for, while loop ->Iteration
  • 递归 Recursion(Divide & Conquer,Backtrace)
  • 搜索Search：深度优先搜索Depth first search, 广度优先搜索 Breadth first search,A+,etc
  • 动态规划 Dynamic Programming
  • 二分查找 Binary Search
  • 贪心 Greedy
  • 数学 Math，几何Geometry

五步刷题法

刷题第一遍
  • 5分钟：读题 + 思考
  • 没有思路直接看解法：注意多解法，比较解法优劣
  • 背诵，默写好的解法
刷题第二遍
  • 马上自己写 ->LeetCode提交
  • 多种解法比较，体会 ->优化，对自己的要求是写下Java和Python版本
刷题第三遍
  • 过一天后，再重复去做题
  • 不同解法的熟练度->专项练习
刷题第四遍
  • 过了一周：反复回来联系相同的题目
刷题第五遍
  • 面试前一周恢复性训练
